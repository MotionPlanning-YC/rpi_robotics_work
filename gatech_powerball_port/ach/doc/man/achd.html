NAME

achd - send ach messages over streams  

SYNOPSIS

achd [OPTIONS...] [serve|push|pull] [HOST CHANNEL]  

DESCRIPTION

Daemon process to forward ach channels over network and dump to files  

OPTIONS

-p PORT,
port
-f FILE,
TODO: lock FILE and write pid
-t (tcp|udp),
transport (default tcp)
-z CHANNEL_NAME,
remote channel name
-u microseconds
transmit period in microseconds (implies -l)
-l
transmit latest frames
-r,
reconnect if connection is lost
-q,
be quiet
-v,
be verbose
-V,
version
-?,
show help
 

FILES

/etc/inetd.conf
Use to enable network serving of ach channels. Use a line like this: '8076 stream tcp nowait nobody /usr/bin/achd /usr/bin/achd serve'
 

EXAMPLES

achd serve
Server process reading from stdin/stdout. This can be run from inetd.
achd pull golem state-chan
Forward frames via TCP from remote channel 'state-chan' on host 'golem' to local channel (a pull from the remote server). An achd server must be listening on the remote host.
achd -r push golem cmd-chan
Forward frames via TCP from local channel 'cmd-chan' to remote channel on host 'golem' (a push to the remote server). Retry dropped connections. An achd server must be listening the remote host.
achd -u 100000 pull hubo state
Forward frames from remote state channel at 10 Hz
 

BUGS

If a connection is lost and reestablished (-r option), some frames may be missed.
